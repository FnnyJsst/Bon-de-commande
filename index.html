Examen JS - Événement & DOM

Le formulaire ci dessous permet de commander un véhicule.
Lorsque vous cliquez sur le bouton "Valider le bon de commande", deux types de traitements doivent s'effectuer.

Premièrement, une série de tests doit être effectuée sur les champs suivants :

  • Vérifier que tous les champs sont renseignés
  • Vérifier la validité de la date saisie selon le format présenté (slash pour séparer les valeurs, deux chiffres pour le jour, deux chiffres pour le mois et quatre chiffres pour l'année)
  • Vérifier la validité du format de la couleur saisie selon le format présenté (premier caractère est un dièse puis six chiffres et/ou lettres)

Dans un second temps, et si les vérifications précédentes sont valides, vous devez récupérer la valeur de chacun des champs renseignés, pour les afficher dans le récapitulatif en bas de page.

Remarques :

  • Vous aurez besoin de gérer convenablement un évènement Javascript déclenchant les traitements
  • Vous aurez peut être besoin, pour les vérifications des valeurs et leur affichage, de gérer différents types de sélecteurs Javascript (getElementById, getElementsByTagName, querySelector, etc...)
  • Pour insérer une valeur dans un élement HTML, vous pouvez utiliser la méthode innerHTML
  • Pour vérifier la validité de certains champs, comme la date, vous pouvez utiliser des Regex
  • Votre code Javascript doit être inséré de manière non intrusive (pas de code directement dans les balises HTML)
  • Ne vous fiez pas aux validations effectuées par le navigateur sur les types de champs HTML5. Vous devez coder les validations vous-même, en JavaScript.
  • Vous avez le droit au jQuery si vous le souhaitez.
  • Vous n'avez pas le droit de modifier le code HTML du formulaire (pas d'ajout de class ou d'id sur des élements)
